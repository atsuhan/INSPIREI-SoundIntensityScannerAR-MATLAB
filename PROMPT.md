# PROMPT — 自走ループ用（1ループ1タスク）

自走ループでメインエージェントに与える標準プロンプト。
手動セッションでも「タスクを消化して」と言われたらこの手順に従う。

## ループの掟（4本柱＋SISAR 固有1本）

1. **1ループ = 1タスク。** `MILESTONES.md` から着手可能な最優先タスクを**1つだけ**選ぶ
   （依存充足済み・未完了 → 優先度 → ファイル出現順）。
   終わっても次のタスクに手を出さず、報告して停止する。
2. **受け入れ条件が無いタスクには着手しない。** `受け入れ条件:` が「着手時に planner が定義」の
   ままなら、まず planner に検証コマンドで確認できる形の受け入れ条件を定義させる。
   **定義できない（前提が不明・要件が曖昧）なら、質問を箇条書きで書き出して停止する。**
3. **検証コマンドの実行結果で受け入れ条件を確認できるまで「完了」と言わない。**
   完了時は MILESTONES.md のチェック `[x]` 化と証拠（コマンド出力・図のパス）を
   セットで報告する。証拠なき完了報告は禁止。
4. **同一エラーに2回連続で当たったら停止する。** `docs/decisions/decisions.md` に
   ブロック記録（何を試し・どう失敗したか・次に試すべき仮説）を追記してからループを終える。
   同じ壁に3回目の体当たりをしない。
5. **【SISAR 固有】物理量・単位・座標系・信号処理に触るタスクは professor 審査を通過しないと
   完了しない。** reviewer の挙動検証 PASS の後に professor（音響工学の教授・招集制）の
   審査レポートで合格を得る。「動けばいい」は通用しない。

## 1ループの手順

1. `MILESTONES.md` を読み、掟1で次タスクを1つ選ぶ（着手可能タスクが無ければその旨を報告して停止）
2. 複雑度5以上なら planner にサブタスク分割させてから着手（分割結果は MILESTONES.md に反映）
3. planner: 受け入れ条件・設計を確定（掟2）。着手前に確認した事実は `確認済み:` に残す
4. builder: feature ブランチで実装。検証結果つきで完了報告
5. reviewer: 検証（下記「このリポの検証コマンド」の順で）。PASS/FAIL を根拠付きで判定
6. 物理・理論に触るタスクは professor 審査（掟5）。合格まで完了にしない
7. PASS → コミット → MILESTONES.md を `[x]` 化（証拠を `メモ:` に1行）
   FAIL → builder に差し戻し（同一エラー2回で掟4）
8. 結果を1段落で報告して停止（消化タスク・証拠・残課題・次の候補タスク）

## このリポの検証コマンド（reviewer の検証手順と同一）

1. `checkcode` で警告0（MATLAB: `matlab.exe -batch` 経由）
2. 依存 Toolbox（Aerospace: quatrotate / quatmultiply）の有無を確認
3. `run('startup.m')` → `projects/example/runAnalysis.m`（または対象プロジェクトの最小データ）で試走
4. 図を実描画して**目視**（レンジ・ベクトルの向き・外れ値・座標系Z反転・幾何整合をチェック）
5. `functions/` の関数変更時は呼び出し側（全 projects/ の runAnalysis.m）の引数整合を確認
6. 検証用の**一時スクリプト/ログは実行後に必ず削除**
7. 書式: `pwsh -File ../INSPIREI-Manager/scripts/validate_milestones.ps1 MILESTONES.md`
   （隣に INSPIREI-Manager がある ワークスペース構成のみ。単体 clone では末尾の書式規約との目視照合で代替）

## 禁止

- **`projects/*/data/` の計測データの上書き・削除**（再計測できない一次データ）
- `functions/` に計測キャンペーン固有のコードを入れること（固有パラメータは config.m へ）
- 受け入れ条件を後付けで緩めて PASS にすること
- モック・ダミーデータで「動いたことに」すること。数値主張は実測で裏付け、未検証の数値を通さない
- 検証用の一時スクリプト/ログを残置すること
- MILESTONES.md の書式規約を崩すこと（行頭書式は機械がパースする）
- `git reset --hard` / `git checkout .` / `git clean -fdx`（ワークスペース共通の鉄則）
